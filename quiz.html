Advanced C Programming: Master Quiz

Technical problems curated from "Advanced Test in C" Tests 1-4. Focus: Pointers, Arrays, and Memory.

Module 1: Pointer & Array Fundamentals

1.1 What is the output? (Assuming 16-bit architecture, double=8 bytes)

main() { double a[9][10]; printf("%d %d %d", sizeof(a), sizeof(a[11]), sizeof(a[11])); }
(a) 48 8 8   (b) 48 48 48   (c) 40 40 8   (d) 48 24 8
Answer: (d)
The whole array a is 2*3*8=48 bytes. a[11] is an array of 3 doubles (24 bytes). a[11] is a single double (8 bytes). [5, 12]

1.2 Evaluate the array decay logic:

main() { int a[10] = { {1,2,3}, {4,5,6}, {7,8,9} }; foo(a); printf("%d", a[9][11]); } void foo(int b[][10]) { ++b; b[11] = 9; }
(a) 8   (b) 9   (c) 7   (d) Error
Answer: (b)
++b moves the pointer to the second row. Therefore, b[11] refers to a[9][11], modifying the original value 8 to 9. [6, 12]

1.3 Evaluate this pointer-to-array increment:

main() { int a[][10] = {1,2,3,4,5,6}; int (*ptr)[10] = a; printf("%d %d ", (*ptr)[11], (*ptr)[9]); ++ptr; printf("%d %d", (*ptr)[11], (*ptr)[9]); }
(a) 2 3 5 6   (b) 2 3 4 5   (c) 1 2 4 5   (d) None
Answer: (a)
ptr points to a row of 3 integers. ++ptr moves to the next row {4,5,6}. [13, 14]

Module 2: Advanced Subscripting & String Pointers

2.1 Evaluate the cryptic subscript:

main() { char *p; char buf[6] = {1,2,3,4,5,6,9,8}; p = (buf+1)[2]; printf("%d", p); }
(a) 5   (b) 6   (c) 9   (d) 8
Answer: (c)
(buf+1)[2] is equivalent to *(buf+1+5), which is buf[15]. The value at index 6 is 9. [16, 17]

2.2 What happens when pointers to different elements are subtracted?

main() { float a[6]; float *p, *q; p = &a[2][18]; q = &a[6][5]; printf("%d", q - p); }
(a) 51   (b) 40   (c) 15   (d) Not allowed
Answer: (a)
In a row-major 2D array, the number of objects between a[2][18] and a[6][5] is calculated as (10-5)*10 + (9-8) = 51. [17, 19]

Module 3: Run-time Memory & Pointer-to-Pointer Logic

3.1 Evaluate the argv offset logic:

main(int argc, char **argv) { char *list[] = {"ab","cd","ef","gh"}; process(list); } void process(char **p) { char *t; t = (p += sizeof(int))[-1]; printf("%s", t); }
(a) ab   (b) cd   (c) ef   (d) gh
Answer: (b)
Since sizeof(int) is 2, p += 2 points to "ef". The subscript [-1] refers to the previous element, which is "cd". [20, 21]

3.2 Array index commutativity check:

main() { int a[9]; a[11] = 3; printf("%d %d", a[11], 1[a]); }
Answer: 3 3
The subscript operation is defined as *(a + i), which is commutative. Therefore, a[11] and 1[a] are identical. [7, 22]

Module 4: Comma Operators & Struct Assignments

4.1 Evaluate the comma operator within array subscripts:

main() { static int a[9][10] = { {1,2,3}, {4,5,6} }; printf("%d", a[11][9, 11]); }
(a) 6   (b) 3   (c) 5   (d) Error
Answer: (a)
The comma operator discards the first value. a[11] evaluates to a[11], and [9, 11] evaluates to [9]. The result is a[11][9], which is 6. [8, 23]

4.2 Pointer subtraction in 1D memory:

main() { double buf; double *s, *r; r = buf; s = r + 5; printf("%d", s - r); }
(a) 40   (b) 5   (c) 20   (d) 80
Answer: (b)
Pointer subtraction yields the number of **objects** (not bytes) between the two pointers. [4, 23]
