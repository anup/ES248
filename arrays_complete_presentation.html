Part 1: Arrays

Understanding the Fundamentals

A Problem We Can't Solve (Yet)

Challenge: Given n numbers, find how many numbers are greater than the average.

Example: 85, 92, 78, 95, 88

Average = (85+92+78+95+88)/5 = 87.6

Numbers > 87.6: 92, 95, 88 ‚Üí Answer: 3

The Problem: We need to see ALL numbers to calculate average, then check each number against it. We can't do this in one pass! We need to store the numbers.

Python: What You Know

PYTHON

print(len(scores))        # 5
scores.append(90)         # Add to end
print(scores[0])          # Access first: 85
scores[2] = 80           # Modify third element

    print(score)
Key Features: Dynamic size, easy operations, flexible!

C: A Different World

C
int scores[5] = {85, 92, 78, 95, 88};

// No len() - calculate manually
int size = sizeof(scores) / sizeof(scores[0]);

printf("%d\n", scores[0]);  // Access first: 85
scores[2] = 80;             // Modify third element

// No "for item in array" - must use index
for (int i = 0; i < size; i++) {
    printf("%d\n", scores[i]);
}

Part 2: Memory

What's Really Happening?

Arrays in Memory: Contiguous Storage

When you declare: int arr[5] = {10, 20, 30, 40, 50};

1000 10
1004 20
1008 30
1012 40
1016 50
Arrays are stored in consecutive memory locations. Each int takes 4 bytes (on most systems).

Why Contiguous Memory Matters

When you access arr[3]:

  1. Start at base address: 1000
  2. Calculate offset: 3 √ó 4 = 12 bytes
  3. Jump to: 1000 + 12 = 1012
  4. Read value: 40
This is why array access is O(1)!
Direct calculation, no searching needed.

Memory: Python vs C

Python Lists C Arrays
Storage References to objects Actual values
Location Scattered in heap Contiguous block
Overhead Extra metadata Just the data
Resizing Can grow/shrink Fixed size
Cache Less cache-friendly Very cache-friendly

Why Track Size Separately?

#include

Part 3: Pointers

The Connection to Arrays

What is a Pointer?

A pointer is a variable that stores a memory address. 
int x = 42;
int *ptr = &x;  // ptr stores the address of x

printf("Value of x: %d\n", x);       // 42
printf("Address of x: %p\n", &x);    // e.g., 0x7fff5fbff8ac
printf("Value of ptr: %p\n", ptr);   // Same as &x
printf("Value at address: %d\n", *ptr);  // 42 (dereference)

& = "address of"
* = "value at"

Arrays ARE Pointers!

This is a fundamental truth in C:

int arr[5] = {10, 20, 30, 40, 50};

printf("%p\n", arr);      // Address of first element
printf("%p\n", &arr[0]);  // Same address!

// These are equivalent:
printf("%d\n", arr[0]);   // 10
printf("%d\n", *arr);     // 10 (dereference)
The array name arr is a pointer to the first element!

Pointer Arithmetic

int arr[5] = {10, 20, 30, 40, 50};

int *ptr = arr;  // Points to arr[0]

printf("%d\n", *ptr);       // 10
printf("%d\n", *(ptr + 1)); // 20
printf("%d\n", *(ptr + 2)); // 30

// These are the same:
arr[i]  ==  *(arr + i)
&arr[i] ==  arr + i
ptr + 1 doesn't add 1 byte, it adds sizeof(int) bytes!

Passing Arrays to Functions

PYTHON
def process(arr):
    # len() works
    print(len(arr))
    
scores = [85, 92, 78]
process(scores)
C
void process(int arr[], int size) {
    // MUST pass size!
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]);
    }
}

int scores[] = {85, 92, 78};
process(scores, 3);
Arrays "decay" to pointers when passed to functions. sizeof won't work!

In C: You Do The Work!

C
#define MAX_STUDENTS 100

typedef struct {
    int scores[MAX_STUDENTS];
    int size;  // Track count manually!
} MarksManager;

// Every operation needs:
// 1. Size checking
// 2. Manual loops
// 3. Error handling
// 4. Pointer management
What's 1 line in Python ‚Üí 10-20 lines in C!

Example: Adding a Score

PYTHON
scores.append(90)

# That's it!
C
int add_score(MarksManager *m, int score) {
    // Check if full
    if (m->size >= MAX_STUDENTS) {
        printf("Array is full!\n");
        return 0;
    }
    
    // Add manually
    m->scores[m->size] = score;
    m->size++;
    return 1;
}

Example: Inserting at Position

PYTHON
scores.insert(2, 80)

# Done!
C
int insert(MarksManager *m, int pos, int score) {
    if (m->size >= MAX_STUDENTS) return 0;
    
    // Shift everything right
    for (int i = m->size; i > pos; i--) {
        m->scores[i] = m->scores[i-1];
    }
    
    m->scores[pos] = score;
    m->size++;
    return 1;
}
Need to manually shift all elements to make space!

Example: Searching

PYTHON
if 95 in scores:
    pos = scores.index(95)
    print(f"Found at {pos}")

# Built-in!
C
void search(MarksManager *m, int score) {
    // Manual loop
    for (int i = 0; i < m->size; i++) {
        if (m->scores[i] == score) {
            printf("Found at %d\n", i);
            return;
        }
    }
    printf("Not found\n");
}
