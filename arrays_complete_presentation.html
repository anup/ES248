Part 1: Arrays

Python Lists vs C Arrays

An Array Problem

Problem: Given n numbers, find how many numbers are greater than the average.

Example: 85, 92, 78, 95, 88

Average = (85+92+78+95+88)/5 = 87.6

Numbers > 87.6: 92, 95, 88 ‚Üí Answer: 3

You know how to solve this in Python using lists. Now let's see how to do it in C with arrays.

Python: What You Know

PYTHON
scores = [85, 92, 78, 95, 88]

print(len(scores))        # 5
scores.append(90)         # Add to end
print(scores[0])          # Access first: 85
scores[2] = 80           # Modify third element

for score in scores:      # Iterate directly
    print(score)
Key Features: Dynamic size, easy operations, flexible!

C: A Different World

C
int scores[5] = {85, 92, 78, 95, 88};

// No len() - calculate manually
int size = sizeof(scores) / sizeof(scores[0]);

printf("%d\n", scores[0]);  // Access first: 85
scores[2] = 80;             // Modify third element

// No "for item in array" - must use index
for (int i = 0; i < size; i++) {
    printf("%d\n", scores[i]);
}

Part 2: Memory

What's Really Happening?

Arrays in Memory: Contiguous Storage

When you declare: int arr[5] = {10, 20, 30, 40, 50};

1000 10
1004 20
1008 30
1012 40
1016 50
Arrays are stored in consecutive memory locations. Each int takes 4 bytes (on most systems).

Why Contiguous Memory Matters

When you access arr[3]:

  1. Start at base address: 1000
  2. Calculate offset: 3 √ó 4 = 12 bytes
  3. Jump to: 1000 + 12 = 1012
  4. Read value: 40
This is why array access is O(1)!
Direct calculation, no searching needed.

Memory: Python vs C

Python Lists C Arrays
Storage References to objects Actual values
Location Scattered in heap Contiguous block
Overhead Extra metadata Just the data
Resizing Can grow/shrink Fixed size
Cache Less cache-friendly Very cache-friendly

Why Track Size Separately?

#include

    int count = 0;     // Actual elements: 0

How Insertion Works in C

Let's insert 50 at position 2 in: [10, 20, 30, 40]

Step 1: Shift Right

// Start from end, shift right
for (int i = size; i > pos; i--) {
    arr[i] = arr[i-1];
}

// [10, 20, 30, 40, __]
// [10, 20, 30, 40, 40]
// [10, 20, 30, 30, 40]

Step 2: Insert

// Now insert at position
arr[pos] = 50;
size++;

// [10, 20, 50, 30, 40]
// size = 5
Important: Shift from END to position, not beginning to end! Otherwise you'll overwrite data.

How Deletion Works in C

Let's delete value 30 from: [10, 20, 30, 40, 50]

Step 1: Find

// Search for value
int pos = -1;
for (int i = 0; i < size; i++) {
    if (arr[i] == 30) {
        pos = i;  // Found at index 2
        break;
    }
}

Step 2: Shift Left

// Shift everything left
for (int i = pos; i < size-1; i++) {
    arr[i] = arr[i+1];
}
size--;

// [10, 20, 40, 50, 50]
// [10, 20, 40, 50] (size=4)
Deletion = Find + Shift Left. Both operations are O(n).

Sorting in C (Bubble Sort Example)

C
void bubble_sort(int arr[], int size) {
    for (int i = 0; i < size - 1; i++) {
        for (int j = 0; j < size - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                // Swap
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}

Python: list.sort() ‚ú®
C: Must implement sorting algorithm yourself!

  4. C arrays: contiguous memory, efficient access

Summary

Arrays

  • Fixed size
  • Same type only
  • No bounds checking
  • Track size manually
  • No automatic resizing

Memory

  • Contiguous storage
  • Direct access O(1)
  • Cache-friendly
  • No metadata overhead
  • Garbage values if uninitialized

Operations

  • No built-in methods
  • Manual loops for everything
  • Pass size to functions
  • Can't return arrays
  • No slicing/concatenation
