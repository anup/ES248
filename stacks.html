Stacks

LIFO — A Restricted Data Structure

ES 248 — Data Structures

A Stack of Plates

Imagine a stack of plates in a cafeteria.

Plate D
← top (most recent)
Plate C

Plate B

Plate A
← bottom (first placed)
  • You can only add a new plate on top — that is push
  • You can only remove the topmost plate — that is pop
  • You cannot pull a plate from the middle without first removing everything above it
LIFO — Last In, First Out. The last plate placed is the first one you take. That is the entire idea. A stack is a data structure with just two operations: push and pop.

Push and Pop

push — add to top

push(A):   [ A ]
push(B):   [ A, B ]
push(C):   [ A, B, C ]

pop — remove from top

pop() → C   [ A, B ]
pop() → B   [ A ]
pop() → A   [ ] (empty)
Notice the order: we pushed A, B, C — but popped C, B, A. Reversed. That is not a coincidence. It is the most fundamental property of a stack.

Let's not worry about how push and pop are implemented yet. Let's see what problems we can solve with just these two operations.

Problem 1 — Reverse a String

Given a string, reverse it. Simple enough — but watch what happens with a stack.

Input:  "HELLO"

Output: "OLLEH"

Algorithm: Push every character. Then pop every character.

push 'H', push 'E', push 'L', push 'L', push 'O'
Stack: [ H, E, L, L, O ] ← top
pop → 'O', pop → 'L', pop → 'L', pop → 'E', pop → 'H'
Result: "OLLEH"
LIFO does the reversal for you automatically. Push everything in, pop everything out — reversed. No index arithmetic, no swapping, no second array.

Reverse a String — Code

Assume we have push and pop available. We don't care how they work internally yet.

void reverseString(char *str) {
    int len = strlen(str);

    // Phase 1: push every character
    for (int i = 0; i < len; i++)
        push(str[i]);

    // Phase 2: pop them back — they come out reversed
    for (int i = 0; i < len; i++)
        str[i] = pop();
}
O(n) time, O(n) space. Two passes — one to fill the stack, one to drain it.

Yes, you could reverse a string with two pointers and no extra space. The point here is to see LIFO in action — the stack gives you reversal for free.

Problem 2 — Next Greater Element

Given an array, for each element find the next element to its right that is greater.

Input: [ 2, 1, 5, 3, 6, 4 ]

Output: [ 5, 5, 6, 6, -1, -1 ]

For 2: next greater is 5.   For 1: next greater is 5.   For 6: none, so -1.

Brute force: For each element, scan right until you find something larger.

for (int i = 0; i < n; i++) {
    result[i] = -1;
    for (int j = i + 1; j < n; j++) {
        if (arr[j] > arr[i]) { result[i] = arr[j]; break; }
    }
}
O(n²). For n = 100,000 that is 10 billion comparisons. There is a better way.

Next Greater Element — Stack Insight

Scan left to right. Use a stack to track elements waiting for their answer.

Invariant: The stack always holds elements in decreasing order (bottom to top). When a new element arrives that is larger than the top, it is the answer for everything it beats.

Array: [ 2, 1, 5, 3, 6, 4 ]

Read 2:  stack empty, push 2.                Stack: [2]
Read 1:  1 < 2, push 1.                    Stack: [2, 1]
Read 5:  5 > 1, pop 1 → answer[1]=5.     Stack: [2]
         5 > 2, pop 2 → answer[0]=5.     Stack: []
         push 5.                            Stack: [5]
Read 3:  3 < 5, push 3.                    Stack: [5, 3]
Read 6:  6 > 3, pop 3 → answer[3]=6.     Stack: [5]
         6 > 5, pop 5 → answer[2]=6.     Stack: []
         push 6.                            Stack: [6]
Read 4:  4 < 6, push 4.                    Stack: [6, 4]
End:     remaining stack {6, 4} → answer = -1.

Next Greater Element — Code

void nextGreater(int arr[], int result[], int n) {
    // Again — just push and pop. We don't care how they work yet.

    for (int i = 0; i < n; i++) {
        // Pop everything the current element beats — it's their answer
        while (!isEmpty() && arr[peek()] < arr[i]) {
            int idx = pop();
            result[idx] = arr[i];
        }
        push(i);   // push index, not value — so we can write result[idx]
    }

    // Anything still in the stack has no greater element
    while (!isEmpty())
        result[pop()] = -1;
}
SolutionTimeSpace
Brute force (nested loops)O(n²)O(1)
StackO(n)O(n)
Each element is pushed exactly once and popped exactly once. Total work = 2n operations = O(n). The stack eliminated the inner loop entirely.

Problem 3 — Balanced Brackets

Your editor highlights unmatched brackets. How does it work?

{ ( [ ] ) }  →  valid

{ ( [ ) ] }  →  invalid — wrong closing order

{ ( [ ] }   →  invalid — missing )

Key insight: When you see a closing bracket, it must match the most recently opened bracket. "Most recently opened" = the last one pushed = top of a stack.

Algorithm:

  • Scan left to right
  • Open bracket ((, [, {) → push it
  • Close bracket (), ], }) → pop and check if it matches
  • End of string → valid only if stack is empty

Balanced Brackets — Walkthrough

Input:  { ( [ ] ) }

Read {   → push    stack: [ { ]
Read (   → push    stack: [ { ( ]
Read [   → push    stack: [ { ( [ ]
Read ]   → pop [    match!    stack: [ { ( ]
Read )   → pop (    match!    stack: [ { ]
Read }   → pop {    match!    stack: []
Stack empty at end → VALID

Balanced Brackets — Code

int isMatching(char open, char close) {
    return (open == '(' && close == ')') ||
           (open == '[' && close == ']') ||
           (open == '{' && close == '}');
}

int isBalanced(char *expr) {
    // Still just push and pop — same two operations.

    for (int i = 0; expr[i] != '\0'; i++) {
        char c = expr[i];
        if (c == '(' || c == '[' || c == '{') {
            push(c);
        } else if (c == ')' || c == ']' || c == '}') {
            if (isEmpty()) return 0;            // nothing to match
            if (!isMatching(pop(), c)) return 0;
        }
    }
    return isEmpty();   // valid only if all opens were closed
}
O(n) time — one pass through the string. O(n) space — at most n/2 opens on the stack at once. No nested loops. The stack eliminates the need to search backward.

Three Problems, Two Operations

We solved all three problems using only push and pop.

We never asked how they work. We treated the stack as a black box — an abstraction.

Now let's build the box.

How do we implement push and pop?

The simplest answer: an array and an integer.

Implementing a Stack with an Array

An array + an integer tracking the top index. That is all.

#define MAX 100

struct Stack {
    int data[MAX];
    int top;        // index of the current top element; -1 means empty
};

Visualizing top after pushing 10, 20, 30:

data: [ 10 | 20 | 30 | ?? | ?? | ... ]

          0    1    2    3

                      ↑ top = 2

Initialization: s.top = -1 means the stack is empty — no element at index 0 yet. The condition s.top == -1 is our isEmpty check.

Stack Operations — Code

void init(struct Stack *s) {
    s->top = -1;
}

int isEmpty(struct Stack *s) {
    return s->top == -1;
}

int isFull(struct Stack *s) {
    return s->top == MAX - 1;
}

void push(struct Stack *s, int x) {
    if (isFull(s)) { printf("Stack overflow\n"); return; }
    s->top++;
    s->data[s->top] = x;
}

int pop(struct Stack *s) {
    if (isEmpty(s)) { printf("Stack underflow\n"); return -1; }
    int val = s->data[s->top];
    s->top--;
    return val;
}

int peek(struct Stack *s) {
    if (isEmpty(s)) { printf("Empty stack\n"); return -1; }
    return s->data[s->top];
}

Push and Pop — Under the Hood

push(10), push(20), push(30)

top=-1 (empty)
push(10): top++ → 0, data[0]=10
push(20): top++ → 1, data[1]=20
push(30): top++ → 2, data[2]=30

pop(), pop(), pop()

pop(): val=data[2]=30, top-- → 1
pop(): val=data[1]=20, top-- → 0
pop(): val=data[0]=10, top-- → -1
Push = increment top, write.   Pop = read, decrement top.   Both are O(1). Always.
Overflow: push when full (top == MAX-1) — writes past the array.   Underflow: pop when empty (top == -1) — reads garbage. Always check.

Recap

  • LIFO — the restriction is the point, not a limitation
  • Reverse a string — push all, pop all. LIFO reverses for free.
  • Next Greater Element — monotonic stack turns O(n²) into O(n)
  • Balanced brackets — "most recent open" = top of stack
  • Array implementation — a struct with data[] and top. Push/pop are O(1).
